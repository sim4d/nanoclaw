# Add-Parallel Skill

并行处理和任务分配能力，让 AI 助手能够同时处理多个任务或调用多个子代理。

## 描述

这个 skill 为 AI 助手添加并行处理能力，包括：
- 🚀 **并行任务执行** - 同时运行多个独立任务
- 🤖 **子代理协作** - 使用专门的子代理处理不同方面
- 📊 **任务分解** - 将复杂任务拆分为可并行的子任务
- ⚡ **性能优化** - 利用并行处理提升效率
- 🔄 **任务编排** - 管理并行任务的依赖关系

## 用法

### 基础用法

```
# 并行执行多个任务
你：用 add-parallel skill 同时：
1. 搜索 Python 性能优化技巧
2. 查找最新 JavaScript 框架对比
3. 获取 Golang 最佳实践

Andy：[同时启动三个搜索任务并汇总结果]
```

### 高级用法

```
# 使用子代理并行开发功能
你：用 add-parallel skill 创建一个待办事项应用，
分别用不同子代理处理：
- 前端 UI（React 子代理）
- 后端 API（Node.js 子代理）
- 数据库设计（SQL 子代理）

Andy：[并行启动三个子代理同时工作]
```

## 功能特性

### 1. 并行信息收集
```
同时搜索多个主题
同时抓取多个网页
同时分析多个文件
```

### 2. 并行代码生成
```
同时生成多个模块
同时编写多个测试
同时创建多个文档
```

### 3. 子代理协作
```
主控代理 - 协调和规划
前端代理 - UI/UX 实现
后端代理 - API 和数据库
测试代理 - 测试用例编写
文档代理 - 文档生成
```

### 4. 任务编排
```
识别任务依赖关系
并行执行独立任务
串行执行依赖任务
合并最终结果
```

## 最佳实践

1. **任务独立性** - 确保并行任务相互独立
2. **合理分组** - 将相关任务放在同一组
3. **结果合并** - 提前规划如何合并结果
4. **错误处理** - 处理部分任务失败的情况
5. **资源限制** - 考虑系统资源限制

## 性能优势

- 📈 **速度提升** - 并行处理可节省 50-80% 时间
- 🎯 **专注度** - 每个子代理专注单一任务
- 🔧 **专业化** - 不同任务使用不同配置
- 📊 **可扩展** - 容易添加更多并行任务

## 限制说明

- 当前环境下的并行主要是模拟和任务编排
- 实际执行速度取决于可用资源
- 某些任务可能存在依赖关系
